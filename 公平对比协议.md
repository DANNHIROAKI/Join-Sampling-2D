# 公平对比协议（SJS-2D）

本文档定义本仓库内建议的模型对比协议，用于避免“只优化某一模型的测试路径”。

## 1. 同步协议

- 所有模型必须使用同一 `repeat_protocol`：
  - `full_e2e`：每次重复都执行 `Reset -> Build -> Count -> Sample`。
  - `shared_build`：首轮执行完整流程，后续重复复用已构建状态，仅执行 `Count -> Sample`。
- 禁止在同一张对比表中混用不同协议。

## 2. 语义一致性检查

- `EnumSampling` 必须返回 `exact count`，否则判为失败。
- 非空 Join 下，`Sample` 必须返回恰好 `t` 条样本。
- 上述检查由 runner 统一执行，对所有方法等价生效。

## 3. 结果汇总

- `run.sh` 会输出：
  - `summary_stats.csv`：全量统计（包含 PARTIAL / NR / FAIL）。
  - `summary_matched_ok.csv`：仅保留三模型（`ours/enum_sampling`、`ours/sampling`、`kd_tree/sampling`）均完全成功的可公平对比点。
- 论文主表建议优先使用 `summary_matched_ok.csv`，避免把不对齐点混入结论。
